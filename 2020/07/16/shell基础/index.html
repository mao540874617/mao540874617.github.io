bash基本功能

别名与快捷键

bash常用快捷键

快捷键 作用
ctrl + C 强制终止当前命令
ctrl + L 清屏,相当于clear
ctrl + U 删除或剪切光标前的命令
ctrl + K 删除或剪切光标后的命令
ctrl + Y 黏贴ctrl+u或ctrl+k的内容
ctrl + R 在历史命令中搜索,按下后就会出现搜索界面
crtl + D 退出当前终端

输出重定向

符号 作用
[命令] &> [文件] 以覆盖的方式吧正确的命令和错误的放到一个文件夹中
[命令] &>> [文件] 以追加的方式吧正确的命令和错误的放到一个文件夹中
[命令]&>>[文件1] 2&>>[文件2] 把正确的内容放到文件一,错误的放到文件2

多命令顺序执行与管道命令

命令格式 作用
命令1 ; 命令2 多个命令之间没有逻辑关系
命令1&&命令2 逻辑与,只有1正确执行,才会正确执行命令2
命令1 | 命令2 命令2 命令1的正确输出作为命令2的操作对象

通配符和其他特殊符号

通配符 作用
? 匹配任意一个字符
. 匹配任意多个字符,可以是0个,即任何内容
[] 匹配中括号中任意一个字符,如[abc]代表一定匹配一个字符,或者是a,或者是b,或者是c
[-] 匹配中括号中的任意一个字符,-代表范围[a-z]代表匹配一个小写字母
[^] 逻辑非,表示匹配中括号中的一个非范围内的字符[^0-9]表示匹配一个不是数字的字符

bash中其他特殊符号

符号 作用
‘’ 单引号,在其中的所有特殊符号都没有特殊含义
“” 双引号,在双引号中的特殊符号,除了$ ` \ 外都没有特殊含义
`` 反引号,在其中的内容都是系统命令,bash会先执行,和$()作用相同
$() 同上一个,引用系统命令
# 在shell监本中,#开头代表注释
$ 用于调用变量的值,需要调用变量name的值时,需要用$name的方式得到变量的值
\ 转义符,在\之后的字符失去特殊意义

bash变量

用户自定义变量,变量设置规则

  • 变量不能以数字开头
  • 在bash中,变量的默认类型都是字符串型,要进行数值运算需要指定变量类型为数值型
  • 变量用等号连接时,两边不能有空格
  • 变量的值如果有空格需要使用单引号或双引号包裹
  • 变量的值中,可以使用\转移符
  • 如果需要增加变量,可以进行变量值的叠加,不过需要用双引号包含”$变量名”或用${变量名}包含.
  • 如果把命令的结果作为变量值赋予变量,则需要使用反引号或者$()包裹
  • 环境变量建议大写,便于区分

变量分类

  • 用户自定义变量(本地变量)
  • 环境变量:这种变量主要保存的是和系统操作环境相关的数据
  • 位置参数变量:这种变量主要用来向脚本当中传递参数或数据,变量名不能自定义,变量作用是固定的
  • 预定义变量:是bash中已经定义好的变量,变量名不能自定义,变量作用是固定好的.

环境变量

  • 环境变量: 这种变量主要保存的是和系统操作环境相关的数据
  • 用户自定义变量只在当前的shell中生效
  • 环境变量会在当前shell和这个shell的所有子shell当中生效.如果把环境变量写进配置文件中,那么环境变量会在所有的shell中生效.

系统常见环境变量

PATH 系统查找命令的路径

位置参数变量

位置参数变量: 这种变量主要用来向脚本当中传递参数或数据,变量名不能自定义,变量作用是固定的

位置参数变量 作用
$n n为数子,$0代表命令本身,$1-$9代表第一到第九个参数,十以上的参数需要用大括号包含,如${10}
$* 这个变量代表命令行中所有的参数,把所有的参数当成一个整体
$@ 这个变量也代表所有的参数,但是每个参数区分对待
$# 这个参数代表命令行中所有参数的个数

预定义变量

预定义变量: 是bash中已经定义好的变量,变量名不能自定义,变量作用是固定好的

预定义变量 作用
$? 最后一次执行的命令的返回状态,如果是0,则上一个命令执行正确
$$ 当前进程的进程号(PID)
$! 后台运行的最后一个进程的进程号(PID)

接收键盘输入

选项 变量描述
-p “提示信息” 在等待read
-t 秒数 read命令会在指定的等待时间内等待用户输入
-n 字符数 read命令只接受指定的字符数
-s 隐藏输入数据,适用于机密信息的输入

环境变量配置文件-简介

环境变量配置文件作用: 环境变量配置文件中主要是定义系统默认环境变量如:PATH,HISTSIZE,PSI,HOSTNAME等

主要的环境变量配置文件

前三个是全局的,后两个是对登录用户有效的

  1. /etc/profile(第一个读取的,优先级最高)
  2. /etc/profile.d/*.sh
  3. /etc/bashrc
  4. ~/.bash_profile
  5. ~/.bashrc

系统默认语言

在 /ETC/sysconfig/il8n 中保存着系统默认的语言环境,编码等信息 如 LANG=”zh_CN.UTF-8”
