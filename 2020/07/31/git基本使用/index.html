git使用

git的简单介绍

git是一种先进的分布式版本控制系统，其特点是根本没有中央服务器，每个人的电脑上都有一个完整的版本库，并且git有强大的分支管理机制，非常适合多人协作完成任务。

git的初始配置

  • 在Linux系统中，git的配置文件保存在~/.gitconfig文件中，执行任何git的配置命令都会自动创建这个文件。

git常用命令

命令 参数 作用
git init 初始化新仓库(版本库,repository)
git clone (URL(地址,https://……) 克隆仓库
git status 查看仓库状态
git diff (file) 可以查看文件被怎么修改了
git add (filename) 提交文件
git commit “注释信息” -m 提交文件到仓库中
git rm [参数] (filename) 删除版本库和项目目录中的文件
–cached 只删除版本库中的文件
git log 查看提交历史记录
git feflog 查看命令历史

git各个区的说明

  1. 工作区 : 是还没有添加到(git add .) 的区域,代表正在操作的内容
  2. 暂存区 : (git add .)之后文件到达的地方,类似游戏的存档点,位于工作区和版本库之间
  3. 版本库 : 可以简单理解成一个目录,在这个目录中的所有文件都可以被git管理起来,每个文件的增改查删都可以被追踪历史和还原

工作区的管理

命令|参数|作用
|:—:|:—:|:—:|
|git clean||用来从工作目录中删除没有被跟踪(tracked)过的文件|
||-n|显示哪些文件将会被删除|
||-f|删除目录中没有被跟踪的文件|
||-df|删除目录中没有被跟踪的文件和文件夹|

暂存区的管理

命令 参数 作用 注释
git add . 提交所有修改和新增的文件
git add -u 只提交修改文件不提交新文件
git checkout – (file) 将（file）在工作区的修改全部撤销 –非常重要，没有它就会变成切换分支的命令
git reset HEAD (file) 将已经提交到暂存区的文件，重新拿回工作区 HEAD表示最新的版本
git ls-files -s 查看暂存区文件列表

‘git checkout –’ 其实是用版本库的文件版本替换掉工作区的版本

版本库管理

命令 参数 作用
git reset –hard 清空工作区和暂存区的改动
git reset –hard (版本号) 恢复到指定提交版本（提前用git log 查看版本号）

分支管理

命令 参数 作用
git branch 查看分支
git branch (name) 创建分支
git switch (name) 新版git,切换分支
git checkout (name) 旧版git,切换分支
git branch -d (name) 删除分支
git merge (name) 合并某分支到当前的分支

分支管理特殊情况

  1. git rebase master 如果在向master合并分支的时候,master已经被别人向前推进了,通过这条命令将分支的提交的移到master的最新点养成习惯
  2. 如果现在正在某分支工作到一半需要临时切换到分支,但是却不想提交工作区,可以用git stash命令将工作区暂时存起来,然后切换分支
    命令 参数 作用
    git stash 暂存工作
    git stash list 查看暂存区列表
    git stash pop (暂存区标号) 弹出暂存区,继续工作

远程仓库

和远程git服务器建立连接

  1. SSH key会保存在 ~/.ssh 中的 id_rsa 和 id_rsa.pub 两个文件中,id_rsa是私钥不能告诉任何人,id_rsa.pub是公钥
  1. 登录github,在Account setting -> SSH Keys -> Add SSH Key -> 黏贴id_rsa.pub内容 -> Add Key

添加远程仓库

添加远程仓库有两种方法:

  1. 在远程和本地已经各自创建了两个仓库,现在想将这两个仓库关联起来

    1. 进入到本地的目标仓库
    2. git remote add origin (ssh地址)

  2. 克隆远程仓库到本地
    git clone (远程仓库地址https或者ssh,推荐ssh)

远程仓库的默认名字是origin

远程仓库多人协助

命令 参数 作用
git remote -v 显示远程库详细信息
git pull 远程和本地分支同名,直接执行拉去远程分支的更新
git pull origin xxx 拉取远程origin主机的xxxx分支与当前分支合并
git push origin (本地分支) 将分支推送到远程库的对应分支上
git push origin –delete branch-name 删除远程分支
git checkout -b branch-name origin/branch-name 在本地创建和远程分支关联的本地分支
git branch –set-upstream branch-name origin/branch-name 建立本地分支和远程分支的联系

多人协助基本策略

  • master 分支是主分支,时刻与远程同步
  • dev 分支是开发分支,时刻与远程同步
  • bug 分支是本地修复bug分支,不需要和远程同步
  • feature 分支是开发分支,合作开发一个项目时可以和远程保持同步

多人协作工作模式:

  1. 首先试图使用 git push origin branch-name 推送自己的修改
  2. 如果推送失败,则因为远程分支已经有更新,使用git pull试图拉取更新
  3. 如果拉取中合并有冲突,则解决冲突,并在本地提交
  4. 最后再尝试使用git push origin branch-name推送
